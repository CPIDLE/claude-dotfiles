---
description: 讀取 Claude Code 委派的任務 spec 並執行
---
從 Claude Code 的 memory 目錄讀取任務 spec，按照指示執行。

## Step 1：找到任務檔

讀取當前工作目錄下的 `.opencode-task.md`。

找不到 → 印 `⚠️ 沒有待執行的任務。請先在 Claude Code 執行 /opencode-do <任務>` 然後停止。

## Step 2：讀取並顯示 Spec

讀取 `.opencode-task.md`，完整顯示 spec 內容，然後印：

```
📋 以上是 Claude Code 委派的任務
   Level: L<N>
   準備開始執行...
```

## Step 3：依 Level 執行

### L1（新模組）
- **只新增檔案**，絕不修改現有檔案
- 嚴格按照 spec 的「新增檔案」清單和「API 簽章」實作
- 不新增 spec 未列出的檔案

### L2（受控修改）
- 只修改「可修改檔案」白名單中的檔案
- 絕不動「禁止修改」清單中的檔案
- 按「修改指示」執行，不自行擴大修改範圍

### L3（任務級）
- 自行決定實作結構
- 以「驗收標準」為目標
- 遵守「約束」限制

### 共通規則
- 逐項完成「交付清單」
- 完成後跑測試確認通過
- 不修改 AGENTS.md、CLAUDE.md、任何設定檔（除非 spec 明確允許）

## Step 4：完成報告

全部完成後印：

```
✅ 任務完成
   新增檔案：<列出>
   修改檔案：<列出，若無則省略>
   測試結果：通過/失敗

📋 請回到 Claude Code 執行：/opencode-do review
```

## 重要限制

- **不修改** `~/.claude/` 下的任何檔案
- **不修改** spec 未提及的檔案
- 遇到不確定的地方，寧可少做也不要猜
- 用繁體中文回覆
