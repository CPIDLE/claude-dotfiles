#### F-009
- **位置**: `C:\Users\benth\.claude\plans\greedy-hugging-quilt.md:77`
- **嚴重度**: 🟡
- **問題**: Agent ID 用 `head -c2 /dev/urandom | xxd -p` 產生 4 hex char（2 bytes = 65536 種），同秒碰撞機率約 1/65536。若設計支援 5 個並行 agent，同秒啟動 5 個的碰撞機率 ~0.015%，不高但非零。
- **驗證**: 數學計算：birthday problem with n=5, d=65536 → P ≈ 10/(2*65536) ≈ 0.008%。
- **裁決**: NICE TO FIX — 改為 4 bytes (8 hex char) 可降至可忽略。

## 未覆蓋風險

1. **opencode run 失敗時的 cleanup** — auto 步驟 3 在 exit 非零時停止，但未刪除 `.opencode-output.log`（步驟 6 的清理只在成功路徑執行）。
2. **opencode 超時** — auto 流程沒有定義 timeout。如果 opencode 掛了，Claude Code 會無限等待「完成通知」。
3. **L2/L3 的 auto 模式安全性** — auto 模式自動 approve 所有工具呼叫，但 L2 允許修改現有檔案、L3 允許自行決定結構。auto + L3 = opencode 可以做任何事且自動通過，紅線檢查是否足夠？
4. **無測試套件** — 整個專案沒有自動化測試，examples/ 的驗收全靠手動跑。升級條件要求「測試覆蓋 > 60%」但沒有測試基礎設施。
