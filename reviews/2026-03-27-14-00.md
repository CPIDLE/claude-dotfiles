## 未覆蓋風險

1. **bye.md 的 Google 同步路徑完全失效**：這不只是措辭問題。由於 bye.md Step 7 檢查的 `### Google Doc` 區段已從 progress.md 格式中移除，使用 `/bye`（而非 `/pm bye`）的使用者永遠不會觸發 Google 同步。如果使用者習慣用 `/bye` 而非 `/pm bye`，進度不會同步到 Dashboard Sheet 和 Chat Space。

2. **settings.local.json 白名單失配**：白名單中的 Apps Script URL 使用舊 ID 和 `action=read`，而實際需要的是新 ID 和 `action=upsert`。這意味著 Dashboard 更新的 curl 請求每次都會觸發權限提示，破壞了 pm.md 設計的「自動執行不需確認」流程。

3. **hello.md sync 提示靜默失效**：hello.md 作為 legacy 指令仍被保留且未標示 deprecated（不同於 bye.md 有 legacy 標示），其 sync 提示功能因 progress.md 格式變更而永遠不觸發。

4. **pm-v2-design.md 仍引用 Google Doc 架構**：設計文件（pm-v2-design.md:47, 51, 115, 125, 145）大量引用 Google Doc 概念，與 76f738b 後的 Sheet 架構不一致。雖然設計文件是參考而非執行文件，但作為架構決策記錄已過時。

5. **Apps Script `upsert` action 的 URL 長度限制**：上次審核 F-007 指出 GET URL 長度限制問題。76f738b 新增了 `⚠️ URL 長度限制：Base64 payload 約 8KB 上限` 的警告（pm.md:311），但 Dashboard payload（6 個欄位，每個精簡描述）的 base64 大小遠小於 8KB，此風險已大幅降低。標記為已緩解。
