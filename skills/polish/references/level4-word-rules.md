# Level 4：中文字詞層規則

**用法**：SKILL.md 走到 Level 4 時 Read 本檔。前三層不用管。

**核心**：每條都按「刪除測試」判斷 — 拿掉這個詞，意思變了嗎？沒變就刪。

---

## 1. 清喉嚨（Throat-clearing）

寫了等於沒寫的開場詞，整句砍。

| 砍 | 留 |
|---|---|
| 值得注意的是，X | X |
| 在這裡需要強調，X | X |
| 必須指出的是，X | X |
| 如前所述，X | X |
| 不可否認，X | X |
| 眾所周知，X | X |
| 有趣的是，X | X（除非真的有趣到值得標） |

例外：句子間需要承接時，「不過」「然而」「相對地」是承接詞不是清喉嚨，可留。

---

## 2. 路標詞（Signposts）

「我接下來要說 X」這種告訴讀者結構的，多數可刪。

| 砍 | 改 |
|---|---|
| 首先 / 其次 / 第三 | 改用內容銜接，或全留全砍（不要混用） |
| 接下來，我們將討論 X | 直接討論 X |
| 在本節中，X 將被探討 | X 是…… |
| 綜上所述 / 總而言之 | 通常整段結論句可刪，前面寫清楚就不用總結 |
| 換句話說 | 如果原句寫得清楚就不用換句話說，留著代表前句沒寫好 |

---

## 3. 中文贅字（高頻）

中文最該砍的就是這些，沒有英文對應因此英文 skill 學不到。

| 砍 | 改 |
|---|---|
| 進行 X 的研究 | 研究 X |
| 進行討論 / 進行檢討 / 進行評估 | 討論 / 檢討 / 評估 |
| 做出評論 / 做出回應 / 做出決定 | 評論 / 回應 / 決定 |
| 給予 X 建議 | 建議 X |
| 對 X 進行分析 | 分析 X |
| 相關的 / 相關 X | 多數情況直接刪「相關」 |
| 的部分 | 「美學的部分」→「美學」 |
| ……方面 | 「在效率方面」→「效率上」或刪「方面」 |
| 其實 / 事實上 / 基本上 | 多數情況刪掉句子變更直接 |
| 一定程度上 / 在某種程度上 | 學術可留，其他刪 |
| 所謂的 X | 直接 X，「所謂的」帶輕蔑 tone 慎用 |
| 對於……來說 | 「對台灣來說」→「對台灣」或「在台灣」 |
| 透過……的方式 | 「透過 X 的方式」→「透過 X」 |
| 是一個 X 的 Y | 「是一個複雜的問題」→「複雜」 |
| ……的存在 | 「資源不足的存在」→「資源不足」 |

---

## 4. 評價副詞（Evaluative adverbs）

副詞在誇形容詞但動詞才是工作主力，副詞多半可刪。

| 砍 | 改 |
|---|---|
| 有效地證明 | 證明 |
| 成功地完成 | 完成（「成功」隱含結果） |
| 充分地展現 | 展現 |
| 明確地指出 | 指出 |
| 完整地呈現 | 呈現 |

---

## 5. 空洞強調

加了沒效果的「重」字。

| 砍 | 改 |
|---|---|
| 非常重要 | 重要（或具體說「攸關 X」） |
| 相當顯著 | 顯著（或「比 Y 高 30%」） |
| 極為關鍵 | 關鍵 |
| 十分明顯 | 明顯 |
| 真的很 X | 很 X，或刪「真的很」 |

副詞堆疊（「非常極為十分」）= 寫的人沒把握，砍到剩一個或換具體數字。

---

## 6. 能力詞（Ability phrases）

「能夠 / 可以 / 得以」中文常見冗詞。

| 砍 | 改 |
|---|---|
| 能夠成功完成 | 完成 |
| 可以幫助 X | 幫助 X（語境清楚時） |
| 得以實現 | 實現 |
| 有助於提升 | 提升（「有助於」常是廢話） |

判準：刪掉「能夠」意思變了嗎？沒變就刪。

---

## 7. 文謅謅（Fancy-talk）

賣弄詞彙，不增加資訊。

| 砍 | 改 |
|---|---|
| 蘊含 / 涵蓋 | 包含 / 包括 |
| 構成 / 形塑 | 是 / 形成（看語境） |
| 揭櫫 / 闡明 | 說明 |
| 鑒於 X | 因為 X |
| 抑或 | 或 |
| 倘若 / 倘使 | 如果（學術可留） |
| 是以 / 故而 | 所以 / 因此 |
| 不啻 X | 像 X / 等於 X |

學術文章可保留少量正式詞撐 tone，但同一篇文章保持風格一致 — 不要一句白話一句文言。

---

## 8. 過度讚美 / 過度貶低

中文常見「精彩絕倫」「令人驚艷」「不堪入目」這種放大詞。

判準：拿掉這個形容詞，意思有實質損失嗎？通常沒有，因為形容詞描述「我的反應」不是「對象的特性」。

| 砍 | 改 |
|---|---|
| 精彩絕倫的 X | 寫 X 為什麼精彩（具體 detail） |
| 令人驚艷的 X | 同上，講 detail |
| 慘不忍睹 / 一塌糊塗 | 講具體哪裡爛（誇張詞讀者會打折） |

---

## 9. AI 兵符

LLM 寫東西的招牌動作，看到就砍。

- 「在這個快速發展的世界 / 在現代社會」當開頭
- 三段都用「重要的是」「值得注意的是」開頭
- 段尾「綜上所述」「由此可見」「這顯示了 X 的重要性」
- 「不僅 X，更 Y」這種對偶用得太工整像範本
- 結論段重新講一遍前面講過的（中文叫「廢話結尾」）
- 過多 bullet：連續 3 個段落都用 bullet → 至少改一個成連貫段落
- 「請允許我」「在此」「敬請」過度禮貌

---

## 10. 中文標點

- **句號 vs 逗號** — 中文長句一逗到底是常見毛病，超過 50 字就找位置斷句號
- **頓號 vs 逗號** — 並列短語用頓號（、），並列子句用逗號（，）
- **「：」用法** — 冒號後接的東西要明確，不要「我們發現：很多事情」這種空話
- **書名號《》和引號「」** — 書名章節用《》，直接引述用「」，不要混用

---

## 11. 中文文件慎用英文網路縮寫

繁中正式文件（提案 / 報告 / 簡報 / 信件）裡不要用英文 internet 縮寫當章節標題或正式用語。常見要砍的：

| 砍 | 改 |
|---|---|
| TL;DR | **摘要**（預設）/ 核心 / 重點 / 一頁讀完 / 概要 |
| FYI | 供參 / 給你參考 |
| ASAP | 盡快 / 最遲 X 時前 |
| BTW | 順帶一提 / 補充 |
| TBD | 待定 |
| TBA | 待公布 |
| AKA | 即 / 也就是 |
| ICYMI | 補充說明 |
| FAQ | 常見問題（用作章節標題時改中文）|

**判準**：
- **章節標題**（H1-H4）一律改中文 — 縮寫當標題很跳 tone
- **內文中**若是技術用語（API、SDK、CI/CD、SRE）仍可用英文
- **圈內口語**（聊天、commit message、code comment）不限制

**Blog / 部落格 tone 文件**可保留 1-2 個縮寫做風格點綴，但提案 / 報告類文件嚴格走中文。

1. 先抓 §9 AI 兵符（整句砍最多）
2. 接 §1 清喉嚨、§2 路標詞（整句或整短語砍）
3. §3 中文贅字（最大宗，一個一個換）
4. §4-§7（單詞替換）
5. §8 形容詞驗證
6. §10 標點收尾
7. §11 英文縮寫掃一遍（章節標題優先）

每砍一輪重讀一遍，確認沒改壞意思。
